# 🔴 Crisis Alert Overlay (S_code) 디자인 시스템 사양서 v1.0
## 1. 컴포넌트 목표 및 역할
*   **목표:** 시청자에게 '통제 불능 상태'의 공포를 극대화하고, 즉각적인 해결책(보고서)에 대한 필요성을 강제 주입한다.
*   **작동 트리거:** PoC Widget 내부 지표($RSI$, $MACD$ 등 핵심 변수)가 사전 정의된 **위기 임계치 ($\text{Critical Threshold}$)를 초과하는 순간 (T+5:30)** 발생.

## 2. 시각적 요소 및 디자인 시스템
| 요소 | 상세 스펙 | 애니메이션/인터랙션 로직 | 개발 참고 사항 |
| :--- | :--- | :--- | :--- |
| **배경 오버레이** (Overlay) | 반투명한 검은색 필터 ($\text{Opacity}: 70\%$, $\text{Color}: \#8B0000$ 계열의 어둡고 불안정한 적색조). | 전체 화면에 걸쳐 **불규칙적인 노이즈/글리치 효과**를 오버레이한다. (랜덤한 블록 단위 깜빡임, 주파수 5-10Hz) | `[Animation: NoiseFilter]` 적용 필수. 시청자에게 시스템적 결함을 느끼게 할 것. |
| **핵심 경고 문구** (Primary Alert Text) | "CRITICAL SYSTEM DEVIATION DETECTED" (대문자, 고딕 계열의 날카로운 폰트). | **1초 간격으로 깜빡임 ($\text{Blinking}$)** 효과를 주며, 글자가 나타났다 사라지는 과정에서 데이터 깨짐(Glitch) 애니메이션을 삽입한다. | `[Animation: Blink] + [Effect: DataCorruption]` 로직 구현. |
| **경고 플롯** (Secondary Alert) | "MODEL FAILURE IMMINENT - ACTION REQUIRED" (폰트 크기 축소, 텍스트 흐릿함). | 데이터 그래프 자체가 수평/수직으로 **랜덤하게 흔들리는(Jittering)** 효과를 부여한다. (데이터 무결성 상실 시각화) | `[Animation: Jitter]`. 실제 신호처리 과정에 오류가 발생한 듯 보이도록 구현해야 함. |

## 3. 애니메이션 타이밍 로직 (Keyframe Timeline)
1.  **T+0s (Trigger):** 노이즈 필터가 갑작스럽게 강하게 올라오며 전체 화면을 적색으로 감싼다. (강렬한 사운드와 함께).
2.  **T+0s ~ T+3s:** 경고 문구("CRITICAL...")가 불규칙적으로 깜빡이고, 배경 노이즈와 그래프 지터링 효과가 지속된다. 사용자 시선이 가장 혼란스러운 순간.
3.  **T+3s (Transition):** 위기 상태의 정보 과부하를 잠시 멈추고(1초간 블랙아웃 또는 낮은 명암 대비로 전환), 중앙에 CTA 영역을 강제 배치한다.
