# 🌐 Premium Monitoring Service: State Transition & Interaction Flow Chart v3.0

## 🎯 목표 정의
본 플로우 차트의 목적은 시스템의 모든 상태 변화(State Transition)를 사용자에게 명확히 인지시키고, 특히 **위기 감지 $\to$ 패닉 유도 $\to$ 해결책 제시 $\to$ 안도감 부여**라는 스토리텔링을 시각적/상호작용적으로 완성하는 것입니다.

## 🎨 핵심 비주얼 시스템 정의
| 요소 | 상태 (State) | 색상 코드 (Hex) | 애니메이션 속성 | 연출 지침 |
| :--- | :--- | :--- | :--- | :--- |
| **배경/로그** | 정상 (Stable) | `#1A2333` (딥 네이비) | 평온하고 부드러운 스크롤링. | 녹색 계열의 성공 로그만 점멸하며, 낮은 BPM의 앰비언트 사운드가 깔림. |
| **경고** | 불안정 (Warning) | `#FFC107` (강황/노랑) | 미세한 깜빡임(Pulse), 속도 증가. | 경고 로그가 반복적으로 등장하며, UI 전체에 옐로우 오버레이 필터가 약하게 적용됨. |
| **위기** | Critical Failure | `#D32F2F` (강렬한 레드) | 급격한 깜빡임(Flashing), 화면 떨림(Shake). | **Crisis Alert Overlay 활성화.** 모든 로그 출력 속도가 비정상적으로 빨라지고, 붉은색으로 강제 오버라이드. |
| **해결책** | Intervention/CTA | `#4CAF50` (성장 녹색) | 부드러운 Fade-in, 명확한 포커스 이동. | 위기 상황이 최고조에 달했을 때, '보고서 구매' 버튼 영역만 밝고 안전하게 강조됨. |
| **복구** | Recovering | `#607D8B` (회색-블루) | 점진적인 안정화(Decay), 부드러운 트랜지션. | 붉은 경고가 회색으로 서서히 진정되며, 'System Self-Correction' 메시지가 크게 표시됨. |
| **최종** | 정상 복구 (Stable) | `#2E7D32` (짙은 녹색) | 부드러운 Fade-out, 안정적인 루프. | 모든 경고 오버레이가 사라지고, 시스템이 'Optimal State'에 도달했음을 확신시키는 시각적 여유 공간 확보. |

## 🔄 플로우 차트: 상태 변화 및 상호작용 상세 정의 (V3.0)

### Phase 1: 안정화된 정상 작동 (Stable Baseline)
*   **시나리오:** 시스템이 최적의 환경에서 데이터를 처리하는 모습.
*   **로그 출력:** `[INFO] Data Stream OK. Latency: 12ms.` 등의 메시지가 일정한 속도로 출력됨.
*   **애니메이션:** 로그 컨테이너는 부드럽게(Easing Curve: Ease-in-out) 스크롤되며, 배경은 차분하고 깊이 있는 네이비 계열을 유지함.
*   **사용자 경험:** 사용자에게 '현재 시스템은 완벽하게 작동 중이다'라는 무의식적인 신뢰감을 조성하는 것이 목표입니다.

### Phase 2: 위기 임계치 도달 (Critical Failure Trigger)
*   **발동 조건:** PoC Widget이 예측 범위를 벗어난 데이터를 감지하거나, Mock WS에서 에러 스트림을 수신했을 때.
*   **Step 2a. 경고 발생 (Warning):**
    *   **시각 효과:** 로그 배경색이 `#1A2333` $\to$ `#FFC107`로 급격히 변경됩니다.
    *   **애니메이션:** 화면 전체에 가벼운 노란색 오버레이가 덮이며, 깜빡임(Pulse Rate: 500ms 간격)이 시작됩니다. 로그 출력 속도가 **정상 대비 2배 증가**하며 `[WARN] Deviation Detected...` 메시지가 반복됩니다.
    *   **사운드:** 낮은 주파수의 '삐-삑' 하는 경고음(Low Pitch Warning Tone)이 규칙적으로 울립니다.

*   **Step 2b. 위기 확정 (Critical Failure):**
    *   **발동 타이밍:** 사용자 시선이 가장 많이 집중하는 지점, 또는 데이터의 변동성이 최대치에 도달한 순간.
    *   **시각 효과:** `#FFC107` $\to$ **`#D32F2F` (Red)**로 급격히 전환됩니다. 화면 전체가 강렬하게 깜빡이는(Flashing, 100ms 간격) 애니메이션이 적용되며, 가상의 시스템 로그 자체가 빨간색으로 오버라이드 됩니다.
    *   **Crisis Alert Overlay 호출:** 이 순간 **`assets/design_blueprints/CrisisAlertOverlay_Spec_V2.0.md`**에 정의된 컴포넌트가 화면 중앙 상단에 폭발적으로 나타납니다. (애니메이션: Scale 1.0 $\to$ 1.5 $\to$ 1.0, 속도 150ms)
    *   **로그 출력:** `[CRITICAL] System Integrity Compromised! Immediate Action Required!`와 같은 강렬한 문구가 비정상적인 간격으로 출력됩니다.

### Phase 3: 해결책 제시 및 개입 (Intervention & CTA)
*   **발동 조건:** Critical Alert Overlay가 최고조에 달했을 때, 사용자에게 '스스로 해결할 수 없다'는 불안감을 극대화한 직후.
*   **시각 효과:** 시스템의 모든 요소(로그, 위젯 등)가 순간적으로 **블랙아웃 (Blackout: 50ms)** 되었다가 다시 나타나며 집중력을 강제합니다.
*   **CTA 강조:** 화면 하단에 배치된 '패치 필요 보고서 구매' 영역만 갑자기 `#4CAF50` (성장 녹색)으로 밝게 빛나며, 나머지 불안정한 빨간 배경과 극명한 대비를 이룹니다. (애니메이션: Glow Effect + 줌 인 효과)
*   **사운드:** 경고음이 최고조에 달했다가 갑자기 '뚝' 끊어지면서 **무음 상태(Silence)**로 전환되어, 사용자의 귀와 시선이 CTA 영역으로 집중되게 만듭니다.

### Phase 4: 시스템 복구 및 안도감 (Recovery & Stability)
*   **발동 조건:** 사용자가 $97 보고서 구매를 완료하고, 가상 시스템에 '패치'가 적용된 직후.
*   **Step 4a. 진정 단계 (Recovery):**
    *   **시각 효과:** `#D32F2F`의 강렬한 깜빡임이 점차 빈도와 밝기를 잃으며, 부드러운 회색(`#607D8B`)으로 바뀝니다. 화면 떨림 애니메이션이 사라지고, 시스템 메시지 `[SYSTEM] Self-Correction Protocol Initiated...`가 천천히 출력됩니다.
    *   **애니메이션:** 모든 경고 오버레이가 부드럽게(Ease-out) 축소되며 화면 구석으로 이동하는 듯한 연출을 보여줍니다.
*   **Step 4b. 최종 안정화 (Stable):**
    *   **시각 효과:** 배경색이 `#1A2333` $\to$ `#2E7D32`로 전환됩니다. 모든 경고 흔적이 사라지고, 로그가 다시 `[INFO] System Stabilized. Resilience Index: 98%.`와 같은 안심 메시지를 출력합니다.
    *   **사운드:** 긴장감을 주던 사운드가 점진적으로 잦아들며, '확신'을 주는 낮은 음조의 성공음(Chime Sound)이 재생됩니다.

## ✨ 요약 및 개발 가이드라인 (Action Items)
1.  **애니메이션 원칙:** 모든 상태 변화는 **'급격함 $\to$ 급정지/공백 $\to$ 느린 복구'**의 3단계 흐름을 따릅니다. 감정적 기승전결이 핵심입니다.
2.  **색상 사용 규칙:** 빨강(`D32F2F`)은 **최대 위기**에만, 녹색(`4CAF50`)은 **해결책 구매 순간**과 **최종 안정화**에만 제한적으로 사용합니다. 이 대비가 핵심 판매 포인트입니다.
3.  **로그의 가치:** 로그는 단순히 텍스트를 나열하는 것이 아니라, 시스템이 **'사건 발생 $\to$ 진단 $\to$ 해결 과정'**을 실시간으로 보여주는 하나의 *증거물*로 취급해야 합니다.
