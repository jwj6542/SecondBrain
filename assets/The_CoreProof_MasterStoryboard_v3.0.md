# 🛡️ The Core Proof: 자본 보존 설계 스토리보드 (V3.0)

**최종 목표:** 제품 기능(Feature) 판매가 아닌, 고객의 '자본 보존 확신(Capital Preservation Certainty)'을 판매한다.
**서사 구조:** 공포(Fear/Loss Anxiety) $\to$ 불안정(Stress Event/Failure) $\to$ 안도와 확신(Recovery/Safety).

---

## 🎨 I. 비주얼 시스템 가이드라인 (Design System Rules)

### 1. 컬러 팔레트 규칙
| 상태 | 감정적 의미 | 역할 및 적용 영역 | HEX 코드 |
| :--- | :--- | :--- | :--- |
| **Deep Blue** | 신뢰, 전문성 | 배경, 주요 타이포그래피 (기준 색상) | `#003366` |
| **Safety Green** | 안도, 보호, 성공 | 시스템 정상화, 자본 보존 성공 지표, CTA 버튼 | `#4CAF50` |
| **Warning Red** | 공포, 위험, 손실 | 스트레스 이벤트 발생 시점 (최대 드로우다운), 경고 메시지 | `#D32F2F` |
| **Neutral Grey** | 객관성, 데이터 | 일반 그래프 배경, 텍스트 설명 | `#E0E0E0` |

### 2. 타이포그래피 및 레이아웃 규칙
*   **폰트:** Inter (혹은 유사한 가독성이 높은 산세리프 계열)
*   **레이아웃:** 모든 섹션은 왼쪽 정렬의 명료한 데이터 중심 구조를 유지하며, 시선이 **'빨강(위험)'에서 '초록(안정)'으로 이동하는 흐름**을 강제해야 함.

---

## 🎬 II. 스토리보드 분할 및 애니메이션 스크립트 (Storyboard Sequence)

### A. 섹션 1: Pain Point 정의 (공포 유발 - Fear)
*   **목표:** 고객의 현재 시스템에 대한 불안감을 극대화한다.
*   **시각 요소:** 기존 트레이딩 대시보드 이미지 사용 (복잡하고 혼란스러운 그래프, 높은 변동성).
*   **스크립트/연출 (Animation Focus):**
    1.  페이지 로딩 시, 여러 개의 작은 빨간색 '위험 경고' 팝업들이 빠르게 깜빡이며 노이즈를 만든다.
    2.  그래프는 급격히 위아래로 요동치며, 사용자가 통제할 수 없는 듯한 느낌을 준다.
    3.  **핵심 문구:** "당신의 자본은 예측 불가능한 변동성에 노출되어 있습니다." (빨간색 오버레이 텍스트)

### B. 섹션 2: The Core Proof - 시스템 불안정성 시연 (The Crisis - Stress Test Failure Integration)
*   **목표:** 개발팀이 확보한 **'극한 스트레스 테스트 실패 & 복구 과정'**을 증거물로 제시한다. (가장 중요한 부분)
*   **시각 요소:** *실제 데모의 핵심 컴포넌트(RSI 지표 기반)*를 사용하되, 이를 '시스템 아티팩트'처럼 포장해야 함.
*   **스크립트/연출 (Animation Focus):**
    1.  **(0-3초: Stress Build-up)** 그래프가 폭발적으로 하락하는 모습(Warning Red). 화면에 "Maximum Drawdown Event Initiated" 문구가 붉게 깜빡인다.
    2.  **(3-6초: The Failure Point)** 시스템의 주요 지표들이 무너지고, 경고 메시지창이 가득 찬다. 이 순간, 잠시 모든 애니메이션과 색상이 **멈추는(Freeze Frame) 연출**을 통해 '위기감'을 극대화한다.
    3.  **(6-10초: Recovery Activation)** 갑자기 배경에 '안전 설계 프로세스'를 상징하는 흐릿한 초록색 가이드라인이 오버레이 된다. 이와 동시에, 시스템의 지표가 빨간 영역에서 **초록색 안전 구역으로 수렴하기 시작**한다 (Smooth Curve Animation).
    4.  **핵심 문구:** "일반 시스템은 멈춥니다. 자본 보존 설계는 작동합니다."

### C. 섹션 3: 해결책 제시 및 확신 부여 (Resolution - Safety Green)
*   **목표:** 불안 해소와 솔루션을 연결하여 최종적인 신뢰를 구축한다.
*   **시각 요소:** 깨끗하고 정돈된, 대비가 확실한 인터페이스 디자인.
*   **스크립트/연출 (Animation Focus):**
    1.  섹션 전체의 색상이 Deep Blue $\to$ Safety Green 계열로 부드럽게 전환된다. 시각적 안정감을 부여한다.
    2.  핵심 지표(예: 자본 보존 점수)가 **안정적으로 상승하는 애니메이션**을 반복하여 보여준다. (점진적인 성장 그래프).
    3.  **CTA:** "리스크 관리의 패러다임을 바꾸십시오."

## 🌐 III. 플랫폼별 적용 가이드라인

### 1. Pitch Deck Mockup 적용법
*   **포맷:** 슬라이드 전환(Transition)을 이용한 연출에 집중한다. (슬라이드 A $\to$ B: 빨강에서 초록으로의 색상 변화가 가장 중요함.)
*   **화질:** 고해상도 인포그래픽과 폰트만 사용하며, 복잡한 코드는 절대 보여주지 않는다.
*   **흐름 강조:** 각 섹션 간의 전환(Transition) 자체를 하나의 '연출'로 판매해야 한다.

### 2. Landing Page Blueprint 적용법 (Interactive Web)
*   **포맷:** 스크롤 기반 애니메이션(Scroll-Triggered Animation)에 최적화된다.
*   **UX/UI:** 사용자가 스크롤을 내릴 때마다 섹션 A $\to$ B $\to$ C의 서사 흐름이 시간차를 두고 펼쳐지도록 설계한다. (사용자의 행위 자체가 '안도' 경험이 되게 함).
