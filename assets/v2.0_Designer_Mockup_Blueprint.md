# 🚀 Premium Monitoring Service: High-Fidelity Mockup Blueprint v2.0
**작성 목적:** 최종 판매 프레젠테이션 및 랜딩 페이지의 비주얼 컨셉 확정. 코다리님의 RSI 컴포넌트를 중심으로 'Pain $\to$ Solution $\to$ Certainty' 서사를 완성하는 고화질 아셋 가이드라인.

## 🎯 1. 전체 구조와 흐름 (The Narrative Arc)
전체 페이지는 세 개의 명확하게 분리된 섹션으로 구성되며, 각 섹션의 컬러 팔레트와 메시지 톤은 감정적 목표에 맞게 변화해야 합니다.

| 단계 | 목적/감정 | 핵심 시각 요소 | 지배적 색상 (Dominant Color) |
| :--- | :--- | :--- | :--- |
| **Pain** | 경고, 위기, 불안함 | 과거 시스템의 붕괴 그래프 | `#A30000` (Danger Red) & `#2D2D2D` (Dark Gray) |
| **Solution** | 깨달음, 논리적 안전감 | RSI Component (기술 증명) | `#008060` (Teal Green) & `#F5F5F5` (Off-White) |
| **Certainty**| 확신, 신뢰, 제도적 안정성 | 3단계 기능 그리드 / CTA | `#1976D2` (Deep Blue) & `#FFFFFF` (Pure White) |

## 🎨 2. 비주얼 시스템 상세 가이드라인
### A. 컬러 팔레트 정의
*   **Primary Accent:** Teal Green (`#008060`) - 안전, 혁신 기술
*   **Secondary Accent:** Deep Blue (`#1976D2`) - 신뢰, 금융 안정성
*   **Warning/Danger:** Red (`#A30000`) - 경고, 위험 (Pain Section 전용)

### B. 타이포그래피 규칙
*   **Headline (H1-H2):** 굵고 가독성이 높은 산세리프 계열 (예: Pretendard Bold). 감정적 임팩트를 위해 크기 변화를 극대화한다.
*   **Body Text:** 깔끔하고 긴 호흡의 설명을 위한 기본 서체 (Pretendard Regular).

### C. 아이콘 및 이미지 톤앤매너
*   **아이콘:** 모든 아이콘은 라인 아트(Line Art) 스타일을 유지하며, '보험', '방패', '안전장치'를 상징하는 형태로 통일한다.
*   **이미지:** 실제 사람 사진 사용을 지양하고, 추상적이거나 기술적인 데이터 흐름/시스템의 모습을 모션 그래픽으로 대체한다.

## 🛠️ 3. 구현 필수 컴포넌트 매핑 (Integration Points)
1.  **RSI Component Integration:** Solution 섹션 중앙에 배치되며, 이 컴포넌트는 Pain $\to$ Solution 전환 애니메이션의 중심축이 되어야 한다.
2.  **Gradient Separator:** 각 섹션(Pain/Solution/Certainty) 간의 경계는 단순한 수평선이 아닌, **어두운 색상에서 밝은 Teal Green으로 변화하는 부드러운 그라디언트**를 사용하여 시각적 단절 없이 서사적으로 연결해야 한다.
