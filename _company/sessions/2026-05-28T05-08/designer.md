#### Module 1: [Pain Point] 지정학적 충격 시뮬레이션 (The Domino Effect)
*   **시나리오 출처:** Researcher 보고서 (지정학적 리스크)
*   **목표:** 투자자가 '통제할 수 없는 외부 변수'에 노출된 공포를 극대화합니다.
*   **비주얼 요소:** 전 세계 지도를 배경으로 복잡하게 얽힌 파생상품 및 자산 흐름을 보여줍니다. (난잡함 = 위험)

| :--- | :--- | :--- |
| **제목** | Global Liquidity Crisis: 통제 불가능한 리스크의 전이 | 크고 경고적인 폰트, 배경에 미세한 지진계(Seismograph) 노이즈 추가. |
| **메인 시각화** | 여러 자산군(원유-채권-주식-암호화폐)을 연결하는 다수의 추상적인 선들. 이 선들이 갑자기 끊어지고(Break), 빨간색으로 폭발적으로 번지는 애니메이션 연출. | 복잡하게 얽힌 네트워크 다이어그램 (Network Graph). **실시간 데이터가 아닌, '붕괴되는' 이미지를 사용.** |

#### Module 2: [Detection] 금리 변동성 리스크 포착 (The Blind Spot)
*   **시나리오 출처:** Researcher 보고서 (금리 변동성)
*   **목표:** 전통적 지표(VIX, 금리)만으로는 놓치는 '숨겨진' 위험 신호를 우리의 시스템이 어떻게 감지하는지를 보여줍니다.
*   **비주얼 요소:** 평온해 보이지만 밑바닥에서부터 균열이 시작되는 고대 건축물 이미지나 그래프를 사용합니다.

| 구성 요소 | 상세 내용 | 디자인 가이드라인 |
| :--- | :--- | :--- |
| **제목** | Beyond VIX: 금리 변화가 자본 구조에 미치는 숨겨진 영향 | 전문성을 강조하는 학술적 폰트와 레이아웃. |
| **메인 시각화** | 일반적인 채권 가격/금리 곡선 그래프를 배경으로 배치합니다. 여기에 우리의 시스템이 계산한 **'Expected Shortfall (ES) 마지노선'**을 투명하고 선명한 보라색 경계선으로 오버레이 합니다. | 그래프의 특정 지점(예: 5년 만기 채권과 단기 파생상품 간 괴리)에서 노란색 플래시가 터지는 애니메이션 처리. |
| **핵심 카피** | "변동성 자체보다, 변동성이 만들어내는 구조적 취약성을 포착합니다." / "전통적인 리스크 측정 모델의 한계를 뛰어넘는 다차원 분석." | 우리의 시스템이 계산한 'ES' 값을 숫자로 크게 강조 (예: $X Billion Exposure). |
| **제어점 (Pivot)** | 일반 트레이더가 그래프를 보는 시선(Level 1)과, 우리가 구조적 취약점을 보는 시선(Level 2)을 시각적으로 분리하여 보여줌. |

#### Module 3: [Recovery I] 암호화폐 급락 대응 시뮬레이션 (The Fire Drill)
*   **시나리오 출처:** Researcher 보고서 (암호화폐/선물 급락) + Developer 데이터
*   **목표:** 가장 충격적이고 직관적인 '청산 공포' 상황을 연출하고, 우리의 시스템이 자동 복구하는 과정을 보여줍니다.
*   **비주얼 요소:** 하락 추세가 멈추지 않는 그래프와 함께 마진콜 경고창을 활용합니다.

| 구성 요소 | 상세 내용 | 디자인 가이드라인 |
| :--- | :--- | :--- |
| **제목** | Max Drawdown Crisis: 청산 공포를 통제된 기회로 전환하는 설계 | 긴박감과 전문성이 결합된 톤앤매너. |
| **메인 시각화** | 그래프가 급격히 하락(빨간색)하다가, 특정 지점에서 우리의 시스템이 개입하면서 매도/매수 포지션을 자동으로 조정하는 과정(청록색 화살표)을 보여줍니다. | 일반적인 청산 리스크를 막는 **'자동 방어막 (Automated Shield)'** 애니메이션 효과 필수. 손실 폭이 제한되는 모습을 시각화. |
| **핵심 카피** | "시스템이 강제 매도를 실행하기 전에, 저희가 먼저 포지션을 조정하여 자본을 보존합니다." / "Liquidation Threshold 미만에서 최적의 복원 지점을 찾아냅니다." | '자동 헤징'과 '포지션 크기 동적 조정'이라는 핵심 기능을 프로세스 플로우 차트로 명확히 제시. |

#### Module 4: [Recovery II] 자본 보존 설계 원칙 증명 (The Insurance Mechanism)
*   **출처:** Designer Memory & Developer 데이터 (최종 복원력 결과)
*   **목표:** 감정적 공포를 논리적인 '보험 구매 행위'로 전환시키는 핵심 페이지입니다. 기능 나열을 완전히 배제합니다.
*   **비주얼 요소:** 안전한 느낌의 구조물, 방패(Shield), 또는 보험 계약서 같은 신뢰감을 주는 비주얼 메타포 사용.

| 구성 요소 | 상세 내용 | 디자인 가이드라인 |
| :--- | :--- | :--- |
| **제목** | Resilience First: 자본 보존을 최우선으로 설계된 투자 구조 (The Capital Shield) | 가장 차분하고, 신뢰감을 주는 톤앤매너. |
| **메인 시각화** | 시간의 흐름(Time Axis) 위에 일반적인 포트폴리오가 무너지지만, 우리의 시스템이 적용된 가상의 자산은 일정한 곡선을 그리며 '최저선'을 유지하는 비교 그래프. 이 최저선을 '자본 보존 마지노선'으로 명명. | **빨간색과 대비되는 안정적인 파란색(Blue)** 계열로 전체 영역을 덮어 시스템의 안전성을 시각적으로 각인시킵니다. |
| **핵심 카피** | "우리는 수익률 자체를 팔지 않습니다. 통제 가능한 '회복력'을 판매합니다." / "블랙 스완 이벤트 발생 후, 최소 자본 보존율 $X\%$를 확보하는 것을 목표로 합니다." | 투자자가 가장 듣고 싶어 하는 단어(Safety, Preservation, Guaranteed)를 반복적으로 노출. |
