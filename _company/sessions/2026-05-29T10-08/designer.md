## 🚨 Crisis Alert Overlay: 최종 디자인 시스템 명세서 (v1.0)

### 🎯 목적
사용자가 PoC Widget에서 **위기 임계치(Crisis Threshold)**에 도달했음을 백엔드로부터 통보받는 순간, 시각적 충격과 불안감을 극대화하여 보고서 구매라는 액션으로 전환시키는 것이 목표입니다.

### 🎨 비주얼 시스템 정의 (Tone & Manner)
*   **핵심 메시지:** "시스템이 정상적이지 않습니다. 지금 당장 외부의 전문 지식이 필요합니다."
*   **컬러 팔레트:**
    *   `--color-danger`: `#FF0033` (강렬하고 피가 도는 경고 레드)
    *   `--color-system`: `#00FFFF` (디지털 느낌의 사이버 청록색 - 안정된 데이터, 또는 UI 강조)
    *   `--color-background`: `#121212` (딥 블랙/다크 그레이 - 화면에 몰입도를 높이고 색상 대비 극대화)
    *   `--color-text-primary`: `#E0E0E0` (밝은 회색 텍스트)
*   **타이포그래피:** Monospace 계열 (예: `Source Code Pro`, `Roboto Mono`)을 사용하여 '시스템 로그'나 '데이터 스트림' 같은 느낌을 부여합니다.

### ⚙️ 상태 변화에 따른 애니메이션 로직 (State-Driven Blueprint)

이 컴포넌트는 FSM의 흐름에 따라 총 3단계의 명확한 시각적 경험을 제공해야 합니다. 각 단계는 CSS와 JavaScript 기반으로 제어되어야 합니다.

#### Phase 1: [Trigger] Critical Alert Flash (위기 감지 플래시)
*   **발생 시점:** 백엔드에서 `STATE_CRISIS_DETECTED` 상태로 전환되는 **0ms 지점**.
*   **목표:** 사용자에게 '지금 뭔가 잘못되었다'는 물리적 충격을 전달합니다.
*   **애니메이션 사양 (CSS/JS):**
    1.  **전체 배경 오버레이:** 200ms 동안 `opacity: 1`의 반투명한 `--color-danger` 플래시가 전체 화면을 빠르게 가립니다. (`background-color: rgba(255, 0, 51, 0.8)`).
    2.  **Glitch Effect:** 플래시와 동시에, 오버레이 중앙에 짧은 시간 동안 (30ms) `text-shadow`를 이용한 디지털 글리치 효과가 발생합니다. (예: `text-shadow: 0 0 5px #00FFFF, 0 -0 5px #FF0033;`).
    3.  **사운드 트리거:** 경고음을 재생하고, 화면의 모든 애니메이션 속도를 일시적으로 느리게(Deceleration) 하여 패닉감을 고조시킵니다.

#### Phase 2: [Sustain] Anxiety Message Display (불안 자극 및 로그 폭주)
*   **발생 시점:** 플래시가 사라진 직후, `STATE_ANXIETY_MESSAGE` 상태로 진입한 **0ms 지점**. (총 지속 시간: 5~8초)
*   **목표:** 패닉을 유지시키고, 현재 시스템의 위험도를 구체적인 '로그' 형태로 제시합니다.
*   **애니메이션 사양 (CSS/JS):**
    1.  **메인 메시지 애니메이션:** 중앙에 "🚨 CRITICAL SYSTEM FAILURE DETECTED" 텍스트가 `typewriter` 효과로 점진적으로 나타납니다.
    2.  **데이터 스트림 시뮬레이션:** 오버레이의 하단 영역(약 20% 높이)을 차지하는 가상의 로그 창을 구현합니다. 이 로그 창에는 의미 없는 데이터 플롯과 경고 메시지(`[WARN] Index Drift`, `[ERROR] Correlation Break`)가 불규칙적인 간격으로, 마치 터미널처럼 빠르게 출력됩니다 (Pseudo-Random Generation).
    3.  **시각적 불안정성:** 배경 오버레이 전체에 매우 미세하고 느린 주파수의 노이즈(Noise) 애니메이션을 적용합니다. 이는 시스템이 '흔들리고 있다'는 무의식적인 느낌을 줍니다. (`@keyframes noise { background-position: 0% 0%, 100% 100%; }`).
    4.  **CTA 강조:** 메시지 하단에 "솔루션은 내부 데이터만으로는 불가능합니다."라는 문구가 지속적으로 깜빡이는 애니메이션을 적용하고, 최종 CTA 버튼이 마치 '유일한 출구'처럼 대비되는 `--color-system`으로 빛나도록 설정합니다.

#### Phase 3: [Transition] Solution Suggestion (보고서 제안 및 전환)
*   **발생 시점:** 사용자가 불안감을 최고조로 느끼는 순간, 또는 일정 시간이 지난 후(예: 6초 경과), 보고서 구매 플로우가 강제 진입할 때.
*   **목표:** 패닉 상태를 '통제된 해결 과정'으로 전환합니다.
*   **애니메이션 사양 (CSS/JS):**
    1.  **경고 메시지 축소:** "CRITICAL SYSTEM FAILURE" 텍스트가 갑자기 작아지고 배경이 어두워집니다. (`scale(0.8)` 및 `opacity` 감소).
    2.  **솔루션 등장 애니메이션:** 오버레이 중앙에 부드럽지만 단호한 애니메이션으로, "✅ **[Solution Required]** Premium Monitoring Service 보고서가 필요합니다." 라는 메시지가 나타납니다. (Transition: Opacity 0 $\to$ 1, Y-axis translation).
    3.  **CTA 활성화:** 이 단계에서만 CTA 버튼(`구매하기`)의 인터랙션이 완전히 풀리며, 사용자가 마우스 오버를 할 때 `--color-system`으로 빛나고 `box-shadow: 0 0 20px #00FFFF;` 효과가 강하게 나타납니다.
