JSON 형식으로 정의합니다.\n\n```json\n{\n  \"event_id\": \"UUID-STRING\",\n  \"timestamp_utc\": \"YYYY-MM-DDTHH:mm:ssZ\", // UTC 기준 필수\n  \"module_origin\": \"SignalGenerationCore | DataIngestionLayer | ...\", \n  \"severity\": \"CRITICAL | WARNING | INFO\", // CRITICAL은 자본 보존 원칙 위반 등 심각한 실패 지점.\n  \"event_type\": \"API_DISCONNECT | DATA_MISSING | LOGIC_FAIL | MARKET_SPIKE\",\n  \"description\": \"발생한 기술적/시장적 문제에 대한 상세 설명.\",\n  \"impact_summary\": {\n    \"asset\": \"XAUUSD\",\n    \"position_size_at_failure\": 0.01, // 실패 시점 포지션 크기 (Lot)\n    \"pnl_loss_magnitude\": -1200.50, // 자본 손실 규모 (정량적 증거)\n    \"capital_preservation_risk\": \"High/Medium/Low\"\n  },\n  \"recovery_action\": {\n    \"automatic_trigger\": true, \n    \"manual_intervention_needed\": false,\n    \"mitigation_strategy\": \"Stop-Loss Trigger | Position Reduction (50%) | Wait\", // 시스템이 취한 행동\n    \"time_to_stabilization_s\": 3600.5 // 정상화까지 걸린 시간 (핵심 KPI)\n  },\n  \"raw_log_reference\": \"(로그 파일의 절대 경로 또는 ID)\"\n}\n```\n\n## 📈 핵심 엔티티: Recovery Metrics (복구 지표)\n이 객체는 시스템 복원력을 증명하는 최종 보고서에 사용됩니다.\n\n```json\n{\n  \"test_id\": \"UUID-STRING\",\n  \"start_timestamp\": \"YYYY-MM-DDTHH:mm:ssZ\",\n  \"end_timestamp\": \"YYYY-MM-DDTHH:mm:ssZ\",\n  \"initial_capital\": 100000,\n  \"max_drawdown_absolute\": 5000, // 최대 낙폭 절대값 (KPI)\n  \"mdrr\": 0.85, // 최대 낙폭 대비 회복률 (Max Drawdown Recovery Rate) - 중요 KPI\n  \"avg_recovery_time_minutes\": 120,\n  \"successful_event_count\": 3\n}\n```\n\n## ✅ 데이터 흐름 요약 (Data Flow Summary)\n*   **[Input]** $\rightarrow$ `Raw Market Data` $\xrightarrow{Validation/Pre-process}$ **[Signal Core]** $\rightarrow$ `Trade Signal` $\xrightarrow{Execute/Monitor}$ **[Recovery Engine]** $\rightarrow$ *IF FAILURE:* `Failure Event Log (JSON)` $\rightarrow$ **[Reporting Layer]** $\rightarrow$ `Story Data Packet (JSON)`\n\n--- \n\n### 🎯 다음 작업 요청 사항\n1.  **데이터 유효성 검증:** 이 스키마를 기반으로, 기존에 확보된 로우 로그 (`c:\Users\user\Desktop\AI EDUCATER\Jabis\REL_Report\Recovery_Event_Log.json`)가 이 계약을 준수하는지 확인하고, 불일치 항목이 있다면 수정해야 합니다.\n2.  **API 통합 테스트:** 정의된 스키마를 기반으로 Mock API를 만들고, 다른 에이전트들이 이를 호출할 수 있는지 작은 단위로 테스트하는 코드를 작성합니다.
